static: 인스턴스와 클래스
(Math클래스 모든 method가 static으로 선언 되어 있다.) 
	클래스 변수🛠️[ 접근제어자 static 타입 변수명; ]
	클래스 메소드🛠️[ 접근제어자 static 리턴타입 메소드명() {} ]
 - 메모리에 올라가서 클래스와 메소드랑 다르게 언제 어디서든 객체 생성 없이 사용할 수 있다.

상속(부모/자식): 🛠️[ public 자식클래스 extends 부모클래스 {} ]
 - 클래스 만들 때 superclass란이 부모클래스 선택란이다.
 - 추상화 abstact
 - 부모클래스에 공통되는 부분을 효율적으로 처리할 수 있다.

오버라이딩:
 - 부모와 다르게 자식의 기능을 바꾸는 것
 - 오버로딩과 다르게 매개변수(입력틀)이 같아야함.
 - @Override : 컴파일러에게 오버라이드 됐다고 알려주는 명령어(컴파일속도개선)메소드마다 사용해야함.

5:
자바의 최상위 클래스 Object:
 - 모든 클래스들을 넣어서 사용할 수 있기 때문에 엄청난 확장성을 가지게 된다.
 - 모든 클래스가 만들어질 때 Object를 상속받아 만들어진다.
 - println() 메소드에 객체를 매개변수로 전달하면 toString 메소드가 자동적으로 동작 한다.

추상화:
 - 부모클래스
 - 🛠️[ 접근제어자 abstract 리턴타입 메소드명(매개변수); ]  기능이 다형성을 통해 거의 동작하지 않기 때문에, 기능을 정의만 하는 것
 - 코드의 재사용성이 증가 
 - 	메소드에 static과 abstract를 동시에 선언할 수 없다.
	클래스에 abstract와 final을 동시에 사용할 수 없다.
	abtract 메소드는 private을 사용할 수 없다.
 - 추상클래스는 인스턴스화를 할 수 없다. 객체를 만들 수 없다.	
	 // 메모리에 올라갈 수 없어서 객체화를 할 수 없는 것.
 - 메소드는 선언부와 {구현부}로 나눌 수가 있는데 선언부만 정의해놓는 것이 추상메소드이다.
 - 추상메소드의 구현부는 상속받은 '자식클래스에서' 오버라이딩@Override을 통해 정의한다.
 - 리턴타입으로는 쓸 수 있다. 객체화만 안 됌.
